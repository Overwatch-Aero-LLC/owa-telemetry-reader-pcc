As a server the communications manager will route data between the serial port connected to a groundstation and any of the client TCP connections. float CCommManager::ComputeDownlinkPercent ( UInt16  NetworkID)

Compute the downlink percentage for a specific network.*/.

Compute the downlink percentage for a specific network. This is the percentage of successfully received data to the amount expected. This function should only be called every 5 seconds or so, to make sure that enough data have been received to make a good computation. If no data have been received then this function will return 100%. If the network does not exist this function will return 0%. Note that this data can also be computed by the ground station and returned in groundstation telemetry.

Parameters

Returns
The downlink percentage from 0 to 100.

SInt32 CCommManager::CreateNetwork ( UInt16  NetworkID)

Creates a network.

Create a new network structure for the passed network address. If successful this function returns the internal index of the network structure. Else -1 is returned. If the network already exists then this function will return the current index, without creating a new network structure. This function is used internally by the communications manager, and is only made public for advanced users. Normally you rely on the communications manager to automatically create the network when a new avionics is detected. If this function is called the communications manager will NOT call the NewNetworkCallBack function when the first packet from the avionics arrives.

Parameters
NetworkIDis the destination address of the network structure to be created.
Returns
The internal index of the network structure. -1 if an error.

void *  Parameter 
)

Register the call back function for new networks.

Whenever the communications manager sees a packet from a new avionics it calls a call back function to alert the application foreground that a new avionics has been found. This function is used to register the call back function that the communications manager will call. The call back function accepts two parameters, the network address of the new avionics, and another parameter which is set through this function.

Detection of a new avionics is done each time the communications manager receives a stream packet from the ground station. A new avionics is detected based on whether or not the network data structure with the corresponding network address already exists. If it does not exist the communications manger will call InitNetwork to create the network data structure and then call the registered call back function to alert the foreground.

Parameters
NewNetworkCallBackis a pointer to the call back function which will be called when a new avionics is detected. The call back function receives as its first parameter the network address of the new avionics. The second parameter is echoed from the second parameter passed to this function.
Parameteris stored and passed to the call back function as its second parameter.
